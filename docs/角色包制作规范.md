# Perchy 角色包制作规范

本文档介绍 `.perchy-pack` 角色包的**文件格式**、**图片规范**、**打包流程**，供你自己制作或委托画师制作商业角色包。

一个合规的角色包应该做到：
- 用户**双击拖动一个文件**就能"安装"
- 不需要重启程序
- 有清晰的作者、版本、许可信息
- Perchy 加载失败会 fallback，不影响其他内容

---

## 一、文件格式速览

`.perchy-pack` **本质就是一个 zip 文件**，扩展名改了一下。内部结构：

```
柴犬合集-v1.0.0.perchy-pack
├── pack.json          ← 必需：清单文件（元数据 + 图片列表）
├── images/            ← 必需：所有猫图放这里
│   ├── shiba_01.png
│   ├── shiba_02.png
│   └── ...
├── preview.png        ← 可选：包封面缩略图（未来店铺列表用）
├── LICENSE.txt        ← 可选：许可协议文本
└── README.txt         ← 可选：给用户的说明
```

### pack.json 结构

```json
{
  "id": "your_studio.shiba_collection",
  "name": "柴犬合集",
  "author": "Your Studio",
  "version": "1.0.0",
  "description": "12 只柴犬桌面伙伴，含各种可爱姿势",
  "license": "Proprietary. 个人使用许可，请勿再分发。",
  "created": "2026-07-18T12:00:00Z",
  "engine_min_version": "1.3.0",
  "image_count": 12,
  "images": [
    "images/shiba_01.png",
    "images/shiba_02.png",
    ...
  ]
}
```

| 字段 | 必需？ | 说明 |
|---|---|---|
| `id` | ✅ | 唯一 ID，推荐格式 `作者ID.包ID`（比如 `hveinblg.shiba`）|
| `name` | ✅ | 显示名称，中文英文都行 |
| `author` | ✅ | 作者显示名 |
| `version` | ✅ | 语义化版本，`大.中.小` |
| `description` | 建议 | 一句话描述 |
| `license` | 建议 | 许可文本 |
| `created` | 建议 | ISO 8601 时间戳 |
| `engine_min_version` | 建议 | 最低要求的 Perchy 引擎版本 |
| `image_count` | 建议 | 图片数量（跟 images 字段对得上）|
| `images` | 建议 | 相对路径列表 |

## 二、图片规范

### 尺寸

**推荐**：宽或高 **200–500 像素**之间。

原因：
- **太小**（<150px）：Perchy 默认最大显示 180×180，太小会被拉伸模糊
- **太大**（>800px）：加载慢，用户切换时会卡
- **500 左右最合适**：现代显示器 dpi 高，500 缩放到 180 显示很清晰

### 比例

**推荐**：接近 1:1（正方形）或 3:4（略高）。

原因：
- Perchy 会按最大边 `PET_SIZE` 缩放，保持宽高比
- 极扁（比如 4:1）的猫在窗口上会横一大条，视觉突兀
- 极高（比如 1:3）的猫头会跑出屏幕（尤其是最大化窗口时）

### 背景

**必须透明**（PNG）。

有背景色的图会带一圈白/黑框，融合不进标题栏。

抠图建议：
- **手绘 / AI 生成**：用 [remove.bg](https://remove.bg/) 或 [Adobe Express](https://express.adobe.com/) 免费在线抠
- **手办 / 实拍**：Photoshop 快速选择工具 + 边缘细化
- **原生透明**：Figma / Krita / Procreate 直接导出透明 PNG

### 抗锯齿

**Perchy 的引擎会自然处理边缘抗锯齿**，但**源图本身的边缘要干净**：
- ❌ 硬边 + 一圈灰色残留（抠图没抠净的痕迹）
- ✅ 平滑过渡到 alpha=0（透明）

**验证方法**：把图放在**深色和浅色两种背景**分别看一眼，都干净说明抠好了。

### 数量

**一个包推荐 8–20 张**。

- 8 张以下：轮换很快就重复，用户觉得单调
- 20 张以上：文件包超 20 MB，下载慢

**Perchy 引擎每 3 分钟随机换一张**，8 张就够 24 分钟不重复，视觉上足够丰富。

### 姿势多样性

好的角色包应该让用户在不同心境下都能"看见自己"。参考内置英短包的分类：

| 类型 | 例 | 何时想看 |
|---|---|---|
| 打招呼 | 挥爪 + 微笑 | 一天开始时 |
| 认真工作 | 敲电脑 / 看书 | 想专注 |
| 放松 | 趴着 / 睡觉 | 累了 |
| 情绪 | 举爪欢呼 / 眯眼笑 | 心情好 |
| 陪伴 | 探头看 / 蹭 | 想被理解 |

**避免**：全是同一姿势换角度（会显得敷衍）。

## 三、命名规范

### 包文件名

**推荐格式**：`{英文包名}-v{版本}.perchy-pack`

```
✅ shiba-v1.0.0.perchy-pack
✅ chibi-cats-v2.1.perchy-pack
❌ 柴犬 (副本).perchy-pack
❌ pack.perchy-pack
```

原因：
- 用户可能同时装多个包，文件名要能区分
- 中文/空格文件名在某些老版本 Windows 上有兼容问题
- 版本号在文件名里可以直观看到"我装的是新版还是旧版"

### 图片文件名

**推荐**：`{英文角色名}_{2位编号}.{扩展名}`

```
✅ shiba_01.png、shiba_02.png、shiba_03.png
✅ cat_wave.png、cat_typing.png、cat_sleep.png
❌ 无标题 (1).png、无标题 (2).png
❌ shiba1.png（编号无补零，排序会乱）
```

Perchy 引擎按文件名字母顺序排序，用两位数编号能保证 `shiba_10.png` 排在 `shiba_02.png` 之后。

## 四、打包流程（用 `create_pack.py`）

Perchy 仓库根目录有个 `create_pack.py`，一条命令就能生成 `.perchy-pack`：

### 基础用法

```bash
python create_pack.py \
    --name "柴犬合集" \
    --author "Your Studio" \
    --images ./my_shibas/ \
    --output shiba-v1.0.0
```

结果：当前目录下生成 `shiba-v1.0.0.perchy-pack`

### 完整参数

```bash
python create_pack.py \
    --name "柴犬合集" \
    --author "Your Studio" \
    --version 1.0.0 \
    --description "12 只柴犬桌面伙伴" \
    --license "Proprietary. 个人使用许可, 请勿再分发。" \
    --preview ./my_shibas/cover.png \
    --images ./my_shibas/ \
    --output shiba-v1.0.0
```

| 参数 | 必需？ | 说明 |
|---|---|---|
| `--name` | ✅ | 中文英文都行 |
| `--author` | ✅ | 作者名 |
| `--images` | ✅ | 存放猫图的文件夹（非递归，只扫顶层）|
| `--output` | ✅ | 输出文件名（会自动补 `.perchy-pack` 扩展名）|
| `--version` | 建议 | 语义化版本，默认 `1.0.0` |
| `--description` | 建议 | 一句话简介 |
| `--license` | 建议 | 许可文本 |
| `--preview` | 可选 | 包封面缩略图（未来店铺展示用）|

打完包后建议**用 zip 工具打开看看**，检查：
- pack.json 里的图片列表跟实际内容一致
- images/ 里没有意外的 `.DS_Store` / `Thumbs.db` 之类的垃圾文件

## 五、许可证怎么写

角色包的**核心商业价值 = 许可条款**。参考模板：

### 个人使用许可（最严格）

```
Copyright (c) 2026 Your Studio. All rights reserved.

This character pack is licensed for personal, non-commercial use only
by the individual who purchased it. You may NOT:

- Redistribute this pack file to others
- Extract the images and share them separately
- Use the images in commercial projects (games, apps, streams, etc)
- Modify and resell the pack

Purchase URL: [your store link]
Support: [your contact email]
```

### 团队 / 商业许可（略贵）

```
Commercial license for up to N users within a single organization.
Redistribution restricted to internal use.

For larger deployments contact [your email].
```

### 免费开源许可（推广用）

```
CC BY 4.0 — free to use, share, and remix with attribution.
Attribution: "Character art by [Your Name] via github.com/HveinBlg/perchy"
```

**建议**：至少准备两个包 —— 一个免费引流包（吸粉），几个商业收费包（变现）。

## 六、用户如何"安装"你的包

发给用户后，他们的操作**只需要 3 步**：

1. 收到 `shiba-v1.0.0.perchy-pack` 文件
2. 拖到 Perchy 的 `assets/pets/` 文件夹（跟其他猫图放一起）
3. 等 10 秒，包里的角色自动加入轮换

**不需要重启程序，不需要解压，不需要看说明书。**

想删除只需要：把 `.perchy-pack` 文件从 `assets/pets/` 里删掉，下次扫描（10 秒后）就消失。

## 七、常见错误

### ❌ 图片是 JPEG 而不是 PNG

JPEG 没有 alpha 通道，就算你用 PS "透明"导出，实际上是白色。用 PNG。

### ❌ 图片文件名带中文 / 空格

大部分情况能工作，但极少数环境下会 `UnicodeDecodeError`。**保险起见用纯英文 + 下划线**。

### ❌ pack.json 里的 images 列表跟实际不一致

`create_pack.py` 会自动生成正确的列表。**别手改 pack.json**，改错了 Perchy 会拒绝加载整个包。

### ❌ 引擎版本要求写反

如果你的包用了 Perchy v1.3.0 才引入的某个新特性，就必须写 `"engine_min_version": "1.3.0"`。老版本引擎会跳过这个包，避免奇怪的兼容问题。

### ❌ 一个包塞太多图片

一个包超过 50 MB 用户下载体验就差了。**要出很多角色的话，拆成主题包**（"柴犬 · 春夏"、"柴犬 · 秋冬"），每包 10-15 张，客单价还能更高。

## 八、快速自检清单

发布之前对着这张单子过一遍：

- [ ] 图片全部是透明背景 PNG
- [ ] 尺寸都在 200–500 px 范围内
- [ ] 命名规范（`{name}_{01}.png`，带前导 0）
- [ ] 数量 8-20 张
- [ ] 姿势有多样性（不是同一动作换角度）
- [ ] `pack.json` 里 name / author / version / description 填全了
- [ ] `LICENSE.txt` 写清楚"能不能商用/能不能再分发"
- [ ] 用测试的 Perchy 实际装一下，看 3 分钟切图是否正常
- [ ] 在**深色**和**浅色**窗口标题栏上都测试过（保证抠图没残留）

---

**做出 3-5 个精品包，你就有了持续变现的基础。** 每个包不用太多角色，但要**质量过硬 + 定位清晰**（比如"专门给养狗人士"、"送女朋友的少女心版"、"程序员桌面版"）。

一份高质量的角色包，卖 19.9-39.9 元，能覆盖你**约稿成本的 5-20 倍**。
