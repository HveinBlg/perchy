# V2EX 发帖：完整版本

V2EX 是**技术社区**，用户对**销售 / 求 Star / 装 X** 敏感度极高。写帖子的原则：

- ✅ **谦虚**（"分享" 不是 "推荐"）
- ✅ **技术含量**（讲怎么做的，讲踩了啥坑）
- ✅ **有价值**（读完能学到东西）
- ❌ **不求 Star**（提一句就够了，别乞求）
- ❌ **不夸自己**（让技术自己说话）

**发布节点**：`分享创造`（`/go/create`）

---

## 三个标题候选（A/B 测试）

### 候选 A（推荐）：**信息型标题**
```
分享一个跟着你切窗口跑的桌面小猫，Windows + macOS 都能用（含 10.15）
```
**优点**：信息完整、直接说清楚是什么、"包含 10.15" 精准打中一部分用户
**评估**：**中性偏好**，被点开概率高

### 候选 B：**技术型标题**
```
一天写完 + 21 个坑：分享一个跨平台桌面宠物开源项目
```
**优点**：暗示有干货（21 个坑）、"一天写完"能激起好奇
**风险**：可能显得**炫技**，V2EX 老用户会警惕
**评估**：**高风险高回报**，看社区当天氛围

### 候选 C：**内容型标题**
```
分享: 让你思念的宠物永远坐在你正在用的窗口上
```
**优点**：**情感锤子**，非技术用户也感兴趣，容易出圈
**风险**：**过度煽情**，V2EX 直男可能觉得矫情
**评估**：**低风险**，播放和讨论都稳

## 帖子正文（完整版）

```
标题：分享一个跟着你切窗口跑的桌面小猫，Windows + macOS 都能用（含 10.15）

------

昨天摸鱼时手痒写了个桌面宠物玩具，做了 Windows + macOS 两个平台版本，
分享一下。

**是什么**

一只坐在你**当前激活窗口标题栏**上的小猫咪。你切浏览器它跳过去，
切 VS Code 它也跳过去。桌面上没有窗口焦点时，它自动隐藏。

灵感来源：小红书上那种"坐在窗口上的小人"图，觉得可以做成实际能跑的东西。

**技术栈 + 踩坑记录**

- Python + PyQt6（macOS 10.15 legacy 版用 PyQt5，通过一个 qt_compat 
  兼容层跑同一份代码）
- Win32 API 通过 ctypes：GetForegroundWindow + DwmGetWindowAttribute
  (EXTENDED_FRAME_BOUNDS) 拿到窗口 rect
- macOS 用 AppKit + Accessibility API (pyobjc)，需要辅助功能权限
- PyInstaller 打包成 exe / .app，GitHub Actions 三个平台 CI 自动构建

踩了 21 个坑，特别有意思的几个：

1. Go 处理图片时预乘 alpha 让透明背景 PNG 里深色像素被误判丢失
2. Qt 里 QLabel 用固定 size + 图片居中，导致最大化窗口时透明填充
   区被夹到屏幕内，宠物看着"沉进"应用内容
3. Windows 11 24H2 起 wmic 默认不装了，老 stop.bat 全靠 wmic 失效
4. PyInstaller 打包版进程名是 perchy.exe 而不是 python.exe，
   stop.bat 之前只找 python 完全杀不掉
5. Git 三方合并时，一个 revert commit 让 feature 分支的功能"消失"了

完整复盘写在项目 docs/项目复盘.md 里。

**代码地址**

github.com/HveinBlg/perchy

MIT 开源。Windows Release 里是 .zip，macOS Release 里是 .dmg。
macOS 10.15 用户单独下 -macos-legacy.dmg。

**顺便请教**

想请教下大家的经验：

- macOS 上想做单例锁（防止用户双开两只猫），除了 pid 文件方式，
  还有推荐的方案吗？我看了下 QLocalSocket，感觉略重
- Windows 上现在依赖用户在 assets/pets 里丢 PNG，或者拖 .perchy-pack
  文件。想过做个应用内的 GUI 让用户点选，但又怕给这么个玩具加太多
  功能过度设计。你们做小工具时怎么把握"最小可用"和"体验完整"的
  平衡的？

——

不求 star，纯分享。有想法或者踩过类似坑的大佬来聊聊。
```

## 帖子发送后的运营

V2EX 上一个帖子发出去后，**前 3 小时决定了它能不能上热帖榜**。

**你要做的**：
1. **每次有人回帖，2 小时内回复一次**（不用长，感谢 + 补充一句）
2. **有人问技术细节**，认真答（V2EX 用户懂技术，糊弄看得出）
3. **有人贴自己项目**，去看一眼然后回一个真诚的评论（互相引流）
4. **不要爬楼求 Star**

**你不该做的**：
- ❌ 每小时刷新看数据
- ❌ 发布后 30 分钟没人回就删掉
- ❌ 有人质疑就吵架（保持谦虚）
- ❌ 私信别人 "帮我看下项目"

## 加分策略

### 1. 帖子里附**一张 GIF 或截图**

V2EX 支持外链图片。**上传到 imgur 或 sm.ms**，帖子里贴 URL。

```markdown
截图：https://sm.ms/xxxx.png
```

**没图的帖子转化率低 40-50%**。

### 2. 帖子发布 24 小时后**如果反响好**，去掘金 / 少数派再发一次

**改改语气 + 加更多技术细节**，掘金那边的开发者比 V2EX 更爱看长文。

### 3. **发帖当天**用 X (Twitter) 发一条英文版

用 [docs/商店文案.md] 里的 Product Hunt 短版，触达国际用户。

## 需要避免的雷区

**V2EX 用户见过太多"独立开发者营销"，非常敏感**。以下几点绝对不能踩：

### ❌ 表情包过多

```
❌ 分享一个我做的桌面小猫喵喵喵 🐱🎉✨💕
```

**减到最少**。V2EX 是文字社区，emoji 只用来分点（比如列表前）。

### ❌ 求 Star / 求赞

```
❌ 希望大家能给个 Star 支持一下！
❌ 觉得有用请点个 Star 🌟
```

**根本不提 Star**。有心人会自己去点。

### ❌ 假谦虚 + 硬广

```
❌ 只是个小玩具啦，希望大家能来试试~
```

**要么真谦虚（"分享一下，写着玩的"），要么直接讲价值（"1200 行代码 21 个坑"），中间的"假谦虚"最遭嫌弃**。

### ❌ 一次发多个平台的相同帖子

**V2EX 有反刷子机制**。发完 V2EX 别立刻转贴知乎 / SegmentFault 一模一样的内容。**改改角度**再发。

## 备用回帖模板

回复各种可能的评论：

### 有人说 "Mac 桌面宠物早就有了"

```
对，这个品类 Live2D / Shimeji 都做了很多年。差异化在于：这个
只跟着 **激活窗口**（不是屏幕固定位置），并且只占标题栏最上一
条，其他桌宠一般占屏幕角落一大块。
```

### 有人问 "Windows 上会不会被杀软报毒"

```
会。exe 没花钱做代码签名，SmartScreen 会拦一下，点"更多信息 →
仍要运行"就能过。签名证书一年 300-500 美元，先看反响再决定要
不要投。
```

### 有人问 "有没有 Linux 版"

```
目前没有。tracker 用了 Win32 和 Accessibility 两套 API，Linux 
的话 X11 / Wayland 是两套完全不同的实现方案（wmctrl / xdotool 
vs Wayland ext-foreign-toplevel-list）。有兴趣做 Linux 版的
话可以 PR，我暴露的 tracker 是一个平台派发的 package，加 
_linux.py 就行。
```

### 有人问 "你怎么变现"

```
目前不变现。开源免费，长期就当作品。如果做起来考虑做**付费的
角色包**（.perchy-pack 格式），引擎永远免费。参考 Aseprite 的
模式。
```

### 有人吐槽 "Python 桌面应用性能太差了"

```
公允说 PyQt6 打包完 30MB 起步，占内存 100-150MB，是有点重。但 
Perchy 是常驻应用，冷启动一次之后就一直在，CPU 占用 < 1%（只
在 30ms tick 时查一下窗口位置）。个人觉得能接受。如果要极致
性能，Rust + Slint 或者 C++ + Qt 更合适。
```

**关键关联指标**：
- 浏览 → 点 GitHub 链接的比例大概 **10-20%**
- 点了 GitHub → **看完 README** 的大概 30-50%
- 看完 README → **下载 Release** 的大概 5-15%
- 下载 → **给 Star** 的大概 1-5%

**举个数**：帖子有 1000 浏览 → 100-200 人到 GitHub → 30-100 人看完 README → 5-15 人下载 → 大概 1-5 个 Star

**Star 数很小很正常**，别灰心。**Release 下载数**才是真正的市场信号。
